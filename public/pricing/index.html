02 正式访谈执行

访谈执行

访谈问题来自《访谈问题汇总.pdf》的正式提纲，下面每个节点都能看到对应问题、用户反馈和产品侧提取信息。

用户画像
用户
医疗法务律师团队及内容运营同学，负责把律师专业能力转化成短视频普法内容。
职业属性
律师本人承担专业表达和可信背书，运营侧承担选题、脚本、拍摄、剪辑和发布。
行为属性
已在抖音、小红书持续做真人出镜普法，单条视频约 3 分钟，流程运行两年多。
核心诉求
律师出差或无法出镜时，内容仍能稳定产出，同时保持律师专业形象和合规边界。
功能偏好
优先要能选律师数字人、写脚本、生成视频、保存作品草稿，再逐步增强脚本、声音、模板和合规能力。
场景还原
Who
医疗法务律师和运营团队共同使用。
When
律师出差、无法集中拍摄，或运营需要快速补齐短视频内容时。
Where
律所内容生产场景，主要面向抖音、小红书等线上平台。
What
围绕医疗纠纷、医患矛盾、赔偿维权等主题制作普法口播视频。
How
从数字人库选择律师形象，进入 AI 创作，填写脚本生成视频，并在我的页面沉淀作品和草稿。
访谈问题
  1. 目前是否在做短视频、图文、直播或公众号？
  2. 一周大概发布多少条内容？
  3. 从选题到发布，完整流程走哪些环节？
真实总结 / 提取信息
  1. 当前主要在抖音、小红书双平台持续产出普法视频。
  2. 真人实拍模式已经运行两年多，说明业务流程真实存在且稳定。
  3. 数字人不是重建流程，而是替换现有流程里的拍摄环节，降低断更风险。
访谈问题
  1. 目前真人出镜做一条视频，完整时间成本是多少？
  2. 哪一项工作最占用团队精力？
  3. 如果成本降下来，希望内容生产频率怎么变化？
真实总结 / 提取信息
  1. 单条约 3 分钟视频，从准备到发布大约需要 30 分钟至 1 小时。
  2. 瓶颈集中在律师出差无法拍摄，以及字幕、降噪、选片段、筛脚本等后期重复劳动。
  3. 用户希望用数字人降低对律师现场拍摄的依赖，让内容在律师不在场时也能继续产出。
访谈问题
  1. 医疗法务内容最容易踩哪些平台或合规风险？
  2. 哪些表达不能出现在脚本或视频里？
  3. 如果产品能自动标记法律引用错误或违规表述，价值大吗？
真实总结 / 提取信息
  1. 风险集中在医疗案例隐私、法律条文准确性和平台敏感表达。
  2. “包赢”“保证胜诉”等绝对化表述需要被识别和规避。
  3. 后续功能应补齐敏感词检测、法律条文校验、医疗内容合规检测、风险分级和自定义规则库。
访谈问题
  1. 脚本目前从哪里来，是运营写还是律师写？
  2. 有没有固定表达模板或内容结构？
  3. 是否需要引用具体法律条文、条款号和案例素材？
真实总结 / 提取信息
  1. 当前脚本仍需要人工准备，后续可以从手写脚本逐步升级到 AI 辅助生成。
  2. 内容结构应沉淀到 AI 模板库，例如开头引入、案例/法条、风险提示、引导互动。
  3. 医疗法务脚本不能只是普通文案，后续要增强法律条文引用、案例素材输入和合规风险标记。
访谈问题
  1. 数字人形象是否需要 1:1 还原律师本人？
  2. 声音是否要使用律师原声或克隆声音？
  3. 医疗法务内容的语气和表现应该严肃，还是更有同理心？
真实总结 / 提取信息
  1. 形象层面需要数字人库承接，公司律师合集比泛化数字人更符合可信感。
  2. 声音层面需要声音库承接，先支持可选声音，后续再增强律师原声和克隆声音。
  3. 表现层面要保留专业、严肃、温和的可控风格，后续补齐口型同步、手势、表情和背景。
访谈问题
  1. 项目预算范围和可接受的投入方式是什么？
  2. 希望多久看到第一条可演示 Demo？
  3. 是否先做 MVP 验证，再决定是否继续投入完整方案？
真实总结 / 提取信息
  1. 访谈将预算、周期、试用和成功标准单独成章，说明投入决策需要有明确边界。
  2. 用户需要先看到可运行 Demo，证明数字人口播能替代真人拍摄的一部分流程。
  3. MVP 应优先跑通选数字人、写脚本、生成视频、沉淀作品草稿的闭环。
访谈问题
  1. 内容最终怎么变现？
  2. 观众看了视频后如何联系你们？
  3. 从看到视频到真正成交，完整路径是什么？
真实总结 / 提取信息
  1. 内容价值落在案源获取和专业信任建立，不只是品牌曝光。
  2. 短视频要承接到咨询、私域或后续沟通链路，所以内容不能只追求播放量。
  3. 后续需要数据看板和发布数据回传，记录播放、点赞、评论等指标，为获客分析做准备。
访谈问题
  1. 如果先做试用版，必须包含哪些内容？
  2. 试用多久，或生成多少条之后判断是否继续？
  3. 判断 MVP 成功的标准是什么？
真实总结 / 提取信息
  1. 第一阶段要先验证数字人视频生成闭环，现有功能可用 AI 创作页、数字人库、声音库和模板库承接。
  2. 试用应围绕真实医疗法务脚本生成多条样片，观察是否能减少真人拍摄依赖。
  3. 成功标准是能稳定从“选人/上传人像、写脚本、生成视频、保存作品”跑通，合规、剪辑、分发、数据看板作为后续增强。

01
律师资产
访谈中的断更问题，本质是律师本人无法持续出镜。首页看板和数字人库先把律所律师整合成可选择、可生产的数字资产。
02
快速开工
用户需要替代真人拍摄环节。AI创作页用“选择数字人或上传照片 + 手写脚本”的方式，先跑通从人到视频的生产闭环。
03
资产沉淀
真人实拍已经运行两年多，后续会产生作品、草稿和复用素材。我的页面承接数字资产管理，避免内容散落。
04
专业形象
访谈担心数字人“像网红不像律师”。数字人库以公司律师合集呈现，让用户从可信律师形象里选择，而不是从泛化形象里挑选。
05
声音信任
医疗法务内容需要严肃、温和、可信的表达。声音库承接音色选择和后续律师声音资产沉淀，为声音克隆增强预留空间。
06
模板控质
访谈明确脚本需要结构化表达。AI模板库承接开头引入、案例/法条、风险提示、引导互动等专业口播模板。

05 需求分层与优先级

需求分层

需求池仍以真实访谈为基础。这里把优先级改成“现有功能先承接，访谈需求再增强”的版本，避免和当前产品能力脱节。

痛点
  • 律师频繁出差，人不在就拍不了
  • 内容产出完全依赖律师本人在场
  • 字幕、降噪、选片段、筛脚本占用大量时间
  • 数字人必须无缝替换现有拍摄流程
  • 法律内容错误会带来合规责任
痒点
  • 律师形象要专业，避免“像网红不像律师”
  • 脚本需要法律条文来源和案例素材支持
  • 合规检测要能区分高/中/低风险
  • 旧内容和脚本模板希望沉淀复用
  • 多平台发布希望减少重复操作
爽点
  • 输入选题关键词即可生成医疗法务脚本
  • 一次提交多条脚本，排队批量生成视频
  • 自动口型同步、手势表情和专业背景
  • 自动字幕、去口误、去废话、去停顿
  • 发布后播放、点赞、评论数据回传
P0
  1. 首页看板：展示律所律师数字人库，并提供定制数字人、制作数字视频入口
  2. 我的页面：沉淀我的数字资产，包括作品、草稿等
  3. AI创作：选择数字人或上传人物照片，手动填写脚本生成视频
  4. 资源库：数字人库、声音库、AI模板库共同支撑“选人、配音、套模板”
P1
  1. AI脚本增强：从手写脚本升级为关键词、热点事件辅助生成
  2. 模板增强：支持案例素材、法律条文、风险提示等结构化字段
  3. 数字人与声音增强：补齐口型同步、手势表情、律师原声和克隆声音
  4. 合规增强：敏感表述、法条校验、医疗隐私和风险分级
P2
  1. 多平台分发：抖音、小红书、B站发布和规格适配
  2. 数据看板：播放量、点赞、评论等数据回传和复盘
  3. 内容资产检索、复用和二次编辑
  4. 把“我的页面”扩展为完整内容资产工作台

07 迭代计划与最终总结

未来规划

规划以《访谈问题汇总.pdf》中的 PRD 为准：第一期只做能验证核心价值的 MVP，后续再扩展克隆、剪辑、分发和数据能力。

MVP 第一期
验证核心闭环
目标是验证“AI 脚本 + 数字人视频 + 基础合规”能否替代真人拍摄环节，解决律师不在场导致的断更风险。
  • AI脚本生成（医疗法务方向）
  • 数字人视频生成（通用形象 + 标准配音）
  • 合规审核辅助（基础敏感词 + 法条校验）
  • 批量生成：一次提交 5 条脚本
  • 交付目标：能跑通从选题到视频初稿的可演示 Demo
V1.1 提效版
补强生产效率
围绕 Excel 需求池中已拆出的 P0/P1，补齐剪辑和专业表达能力，减少运营在后期上的重复劳动。
  • 智能剪辑：自动字幕、AI降噪、去口误废话停顿
  • 脚本增强：法律条文引用、案例素材生成、当事人信息脱敏
  • 数字人增强：基础手势动作、严肃/温和表情、自定义背景
  • 合规增强：风险分级、自定义合规规则库
  • AI图文生成：图文文案、封面/配图、图文合规审核
V2.0 增长版
扩展运营闭环
PDF 明确第一期不包含的能力，可作为下一阶段增长能力：从“能生成视频”走向“能分发、能复盘、能复用”。
  • 律师本人形象/声音克隆
  • 多平台自动分发：抖音、小红书、B站对接与定时发布
  • 发布后基础数据回传：播放量、点赞、评论数
  • 数据看板：对比真人实拍与数字人口播效果
  • 内容资产管理：历史脚本、视频素材检索、复用与二次编辑
